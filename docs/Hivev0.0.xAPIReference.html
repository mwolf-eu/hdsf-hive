Harvest Interactive Visualization Engine

Hive v0.0.x API Reference

The Harvest Visualization Engine uses multiple object formats to define a chart or graph.  The core does no drawing itself, instead relying on an extensible plugin architecture.  Both the core and plugins integrate D3 and draw heavily from a Grammar of Graphics by Leland Wilkinson.

Conceptually, each plugin is as limited as it can reasonably be while still getting the job done efficiently, and each functions in a layered context with other plugins to produce a visualization.

Features

  • Supports a wide variety of chart / graph types
  • HTML flex-like templating worker for unlimited layout options
  • Full text handling
  • Responsive
  • Color schemes
  • SVG exporting
  • Mixed chart types
  • GGPlot2 style facets
  • Hybrid svg / canvas charts
  • Several graph force algorithms
  • Data loading, wrangling, naming
  • Extensible draw object plugin system
  • Rendering using svg, paperjs, threejs
  • Popups (Including cross graph popups)
  • Mouseover change: color, opacity, scale
  • Includes: basic templates, basic wranglers
  • Supports: scales-accessors, gradients, rotation, all d3 axes, guides/legends

Lifecycle Overview

A typical lifecycle of a chart or graph would be as follows:

  • A Hive.Visualization object is instantiated.
  • If the default layout template was not used, it would be created and optionally modified.
  • The config > element section sets the renderer and HTML element attributes and event listeners. Eg: width, height, HiDPI, zoom, panning, etc.
  • The data section is parsed, optionally loading and wrangling data from an url.
  • The frame (layout template) is parsed
  • The accessors are parsed
  • The draw array is executed in order, and generates visuals using the data and scales defined in accessors, and drawing the result to the selected frame node.
  • Resize, mouse, and other events are serviced

Simple Example

Examples can be worth a 1000 manuals.  Below is one of the simplest configurations possible while remaining interesting.

 1   let data = [{x:1, y:2}, {x:2, y:4}, {x:3, y:6}, {x:4, y:4}, {x:5, y:-2}];

 2   let cfg = {

 3      accessors:{

 4        x:{domain:[0,1,2,3,4,5,6], type:'band', range:'width'},

 5        y:{domain:[-3,7]},

 6      },

 7      data:[{content:data}],

 8      draw:[

 9        {name:'axis', scale:'{x}'},

10        {name:'axis', scale:'{y}', opt:{orientation:'left'}},

11        {name:'rectangle', x:'{x}', y:'{y}', v:'0{y}'},

12        {name:'point', x:'{x}', y:'{y}'}

13      ]

14    }

15    new Hive.Visualization(cfg);

Which results in the following chart:

It is important to remember that the configuration has many defaults.  When you don’t see a particular attribute, the default is used.

The breakdown by line is as follows:

  1. The data object array is defined
  2. General configuration starts
  3. Accessor configuration starts
  4. A band scale is made. The domain is 0 to 6.  The range is the width of the frame for the plugin where it is used, and the data row object key is ‘x’.
  5. A linear scale is made. The domain is -3 to 7.  The range is the -height of the frame for the plugin where it is used, and the data row object key is ‘y’.
  6. Accessor config ends
  7. Data content is defined.
  8. Draw configuration starts
  9. The bottom axis is drawn using the defined scale for x
  10. The left axis is drawn using the defined scale for y
  11. Rectangles are drawn for each x/y in the data.  The baseline (v) for the rectangles is bound to the position determined by the value 0 as scaled by the y accessor.   It’s worth noting here that if all y values were positive, the baseline could be simply ‘0’ which coincides with the bottom of the frame.  Since there are negative y values, the baseline needs to extend to wherever the scale places y==0.
  12. Points are drawn for each x/y in the data.
  13. Draw configuration ends
  14. General configuration ends
  15. The engine is called to render the configuration

Instance

In general most code will maintain the visualization object instance by storing the return value of a call.  Ex: “myvis = new Hive.Visualization(cfg)”.  Alternatively, the instance is stored in the child elements of the container in the _visualization property.  Ex: “document.querySelector("#visualization > canvas")._visualization;”

Core Configuration

A visualization configuration contains several sub-configurations which specify (among other things) the drawable space, data, data accessors, drawable transforms, and visual elements. Essentially, all the components that form a visualization.

Root of the config tree (Object)

loglevel: (log, warn, error) Default: warn

A user can specify values of log, warn, or error which work similarly to their JS counterparts. The exception is that to avoid noisy consoles when debugging, output can be limited by severity. Eg: Setting to 'warn' would only print warnings and errors, thus prohibiting log. Plugins should rely on these rather than printing directly to the console.

name (String) Default: A runtime generated unique string/number combination

A unique id for your graph. Currently used for element id's and to disambiguate event data for grouped graphs.

onStateChange (function) Default: none

A handler function for state changes within the visualizer.

Current argument values passed to the hander:

“PARSE_CFG_END” - All lines in the configuration are parsed and created.

'DRAW_END' - All lines in draw have been run

'FRAME_CHANGED' - A template node geometry has changed (see Templates)

 'PARSE_THREE_SVG_END' - Three.js renderer has finished creating all objects (lines/meshes).

element (Object)

The element configuration handles all attributes of the element that is being drawn to and its associated renderer.

selector (String) Default: "#visualization"

The containing element for the visualization.

exportName (string) Default: "visualization"

The name of an exported and downloaded file.

style (object) Default: none

Key-value styles applied to the containing HTML container object (usually a div).

attrs (object) Default: none

Key-value attributes applied to the containing HTML container object (usually a div).

zoom (boolean) Default: false

Enables zoom via mouse wheel.

drag (boolean) Default: false

Enables pan via mouse dragging.

popup > enabled (boolean) Default: true

Currently, popup functionality relies on Tippy for popup management. If the library requirements are satisfied and this attribute is enabled, tooltips will be available to plugins.

renderer > name (paperjs, svg) Default: paperjs

Specifies the backend renderer for the visualization.  The core supports one of each render type (hybrid rendering) when two separate visualizations using different renderers are called on the same element selector.  

As a heuristic, it can be assumed that canvas based renderers (paperjs) are less flexible and require larger external libraries as compared to DOM (SVG) approaches, but may be faster and use much less memory when rendering large amounts of data.  That said, in certain cases (currently panning & zooming) the SVG optimizations in the Chrome browser are more performant providing a smoother experience than in the latest PaperJS.

renderer > hidpi (boolean) Default: false

For the paperjs renderer canvas hidpi be turned on resulting in a canvas that is double the specified size and invoking an upscaling algorithm which can blur the visual.  Warning:  This will interfere with popup placement and readability.

frames (Object)

A frame object defines a visual layout via nested HTML flex-like bounding boxes with margins, padding, et.al. Facebook Yoga is used to parse the object and return bounding box geometry.

To write new templates, refer to the links below for implemented attributes and arguments:

Function Declarations with arg types and return values

https://github.com/facebook/yoga/blob/master/javascript/sources/entry-common.js#L123 

Enums

https://github.com/facebook/yoga/blob/master/javascript/sources/YGEnums.js 

Frame attributes and Yoga methods are very similar except that frame attributes are lower case, not camel-case and both the prefix “set” and suffix “percent” are removed.  Attribute values (Enums) are lower case and use - instead of _.  Eg: Yoga ALIGN_CENTER becomes align-center

Ex:

Yoga setFlexDirection becomes flex-direction.

If a number is assigned the value is px Eg: height:50 makes height 50px

If a string is assigned the value is a percent  Eg: height:”50%” makes the height 50% of its parent container

The Yoga methods for the above would be setHeight(50) and setHeightPercent(‘50’) respectively.

In addition to Yoga values the following are available to every node in the object:

triggerStateChange: Set to true to be notified that the geometry for this node has changed

user: Arbitrary data returned to the onStateChange handler

opt > rotate: Sets the rotation of the graph. Currently only the 4 cardinal directions are supported. This can also be passed when selecting template configurations. Eg:

vis.templates.chart({viewData:{rotate:90}});

For a visualization of a new or existing template see “Flextool” below.

accessors (Object)

Accessors are scaled references to a value in a row of data. Most often the attribute name (x in for example) corresponds to the key name in a data row. When that is not desired (second example), "field" can be specified (see below) and an arbitrary label used for the accessor attribute.

field: (String) Default: none

Explicit override of the attribute name for a data row.

domain (Array) Default: [0,1]

The input bounds of the scale.

type (String) Default: 'linear'

The scale type. All D3 scales are supported: https://github.com/d3/d3-scale

range (Array or (width | height | -height | circleRadians)) Default: -height

The output bounds of the scale. Scaled accessors are resolved per frame.  This means width, height, and -height will always be with respect to the bounding box of the draw object’s frame. circleRadians will convert to [0, 2].

Interpolate (Function) Default: none

A handler function to expose d3 scale interpolation for building a custom domain to range mapper.

interpolator (Function) Default: none

Exposes similar functionality as interpolate but when the scale type is sequential.

Unknown (Any value scales accept): Default: none

Sets the output value of the scale for undefined or NaN.

prefilter  (Function) Default: none

A value handler which can modify a pre-scaled value.

postfilter (Function) Default: none

A value handler which can modify a post-scaled value.

data (Object Array)

A data object attaches an arbitrary and unique name to each dataset used.

name (String) Default: 'data-0'

The datset name.

content (Object Array) Default: none

The dataset.  Most often, datasets are in long format.  If however the associated plugin creates a single object from the data (Eg: line, area, violin), a packed format may be used where several long formatted data are encapsulated in a single array.  Eg: [[data],[data],[data]]

url: (string) Default: none

A url to data to load.

handler: (function) Default: none

A user function that receives the data, optionally wrangles it, and returns an array which will be assigned to “content”.

draw (Object Array)

The draw object contains an array of plugin configurations. Plugins are executed in-order to the specified frame. (see plugins below)

Draw directives have typically three sections: root, attr, and opt.  At the root of the object will be all the parameters that are needed by the generator function. (Eg: area, line, text, etc.) These parameters are detailed in the plugins section.

Currently all generator methods which accept handlers, all attribute values, and opt > text > (textAnchor, rotation, pad) can be defined using a scaled accessor. (see Parameter Values)

Common parameters:

{draw object root}

D3 shape generators are currently used for arc, area, axis, binhex, geopath, line, link, and symbol (called point). Changes and additions to the generator options are detailed in the plugins section below.  Otherwise refer to the D3 documentation here: https://github.com/d3/d3-shape 

name: (string) Default: none

The name of the generator function.

data: (string) Default: The first data in the data section array

The data bound to the draw directive.

frame: (string) Default: Plugin specific 

The flex template selector where objects should be drawn.  The selector supports css-style whitespace and “>” operators.  

A simplified guide title example:

{id:”guideTitle”, attr:{'align-items':'align-flex-start'}, height:25, children:[

          {id:'title', attr:{'justify-content':'justify-flex-end', 'align-items':'align-flex-start'}, children:[

            {id:'text', attr:{}          

          ]}

        ]}

In the above template, a text anchor “text”, is a child of the “title” container which is a child of the “guideTitle” container.  To draw text in the correct location, a  “label” draw directive needs to specify the text anchor child.  As with css selectors there are multiple ways to form the selector.  A less specific way would be “guideTitle text”.   A better choice would be “guideTitle > title > text”. In the real world multiple guide titles would each have a unique prefix and therefore a unique selector to target each text anchor.

attr: (object) Default: Plugin specific

An HTML attr:value pair applied to each object.  Eg: fill:’blue’.  In the case of nested objects (currently axis only), the key is a combined selector and attribute.  Additionally, color can define a vertical gradient with the form: [{'stop-color':'red'},{'stop-color':'yellow'}]

opt: (object) Default: none

Contains generator meta, event (ev), and other plugin specific configuration.

Common Plugin Attributes and Values

Parameter Values

These values called “complex types” come in several forms, and are mostly used to provide values based on the current row of data.  While some combinations of values contained by diacritical marks are allowed, [...] will always refer to a data column and {...} will always refer to an accessor.  When a value has no enclosure it is a literal.

Examples:

42                 is a number literal

“rgb(1,2,3)”        is a string literal

“[color]”        is a column name

“{y}”                is an accessor name

“[lat]{latitude}”        is a generated accessor* name, used to resolve the column lat (geopath only)

“1{color}”        is an accessor name, given the domain input argument 1

* Accessors can be generated automatically by a draw directive. This is currently only used in geopath (maps) for use by subsequent directives. Eg: By point to place icons on a map.

Width Type Attributes

All plugins that support widths, jitter, notches, or whiskers define width so:

When < 1 it is a fraction of the underlying scale bandwidth

When > 1 it is a literal value in pixels

Clipping Attribute

opt > clip (bool) Default: Plugin Dependent

Specifies if the current draw line should be clipped to the bounding box if it overflows.

Text Attributes

If opt > text exists, the text module is loaded and apply the following constraints to reformatting all the text for the given plugin. (Currently axis, and label)

Since the bounding box width and height anchors for text are usually both 0, x and y are usually left to their default values.  That is, in most cases the text location is set by the template and not the x, y configuration here.

opt > text > x (String) Default: 0

Cartesian x

opt > text > y (String) Default: 0

Cartesian y

opt > text > width (String) Default: none

Width in px of the text

opt > text > height (String) Default: none

Height in px of the text

opt > text > lineSpacing (number) Default: 0

The space in px between lines

opt > text > textAnchor (String) Default: 'tm'

Where the resulting text bounding box should be anchored to the x, y point.  In the x dimension there is left, middle, right.  In the y dimension there is top, middle, bottom.  Therefore to anchor text by the bottom right the value would be “br”.

opt > text > pad (number) Default: 0

Padding in px from the first dimension in the anchor.  Eg: “tr” would apply y padding from the top, while “rt” would apply it to the right.

opt > text > yOffset (number) Default: 0

HTML allows for minimal introspection into the exact proportions of a font’s ascenders and descenders.  This value modifies the vertical padding for improved placement.

opt > text > justify (String) Default: 'left'

Text justification

opt > text > ellipseize (bool) Default: true

If the text is truncated by exceeding the height, this specifies whether the overflow will simply be cut off or end in “...”. Read as ellipse-ize.

opt > text > rotate (number) Default: 0

Rotate the text relative to its origin

opt > text > format (function) Default: d=>d

Text formatting accessor

Event Attributes

Where noted, some plugins support events.  The event attributes allow for popups and changing how the element is rendered.

opt > title (String) Default: context dependent

The title used in popups

opt > ev > group (String) Default: none

This is the pubsub message string that triggers popup events

opt > ev > onMouseEnter | onClick > popup > vals (Array) Default: none

The list of values to send to the popup handler for display

opt > ev > onMouseEnter | onClick > popup > handler (Function) Default: Context Dependent *

This function will create HTML content for the popup.  A few convenience functions have already been defined:

Hive.popup.formatLong - Prints color swatch by each value

Hive.popup.formatLongTitleColor - Prints single swatch before the title

Hive.popup.formatWide - Prints data from multiple charts in one popup

* formatLong for events for single elements, formatWide when multiple are triggered by a shared group

opt > ev > onMouseEnter | onClick > element > color (String) Default: none

Changes color of eventing element

opt > ev > onMouseEnter | onClick > element > opacity (Number) Default: none

Changes opacity of eventing element (0-1)

opt > ev > onMouseEnter | onClick > element > scale (Number) Default: none

Changes size of eventing element

opt > ev > onMouseEnter | onClick > element > handler (Function) Default: none

Function called when event triggered

Note: While element > * can be specified for onClick, they currently only make sense for onMouseEnter.   onClick should be used for function and popup triggering.

Guide Attributes

If a draw directive uses data and scaled accessors, it can optionally include a “guide” key at its root to draw the legend.  Supported guides are: point shape, point size, fill, stroke.  The last two are supported for continuous and non-continuous scales.

Guide is an array which can contain:

  • key - A string or array of draw keys using scaled accessors: Eg: [“attr.fill”]
  • title - The title for the guide group
  • qty - The length of the range guide in icon-sized units.  If the icon (set in templates) is 15px and the qty is 4, the resulting range is 60px long.
  • frame - The guide container selector  Eg: 'basic guide-right'
  • labelOpt - An object to be merged into the draw directive for the label text  Eg: {opt:{text:{yOffset:-4}}}
  • titleOpt:Eg: - An object to be merged into the draw directive for the title text {opt:{text:{yOffset:-1}}}
  • itemsWidthMax - Max width (px) of items before wrapping
  • itemsHeightMax - Max weight (px) of items before wrapping

bandwidth (Complex Type) Default: none

The fraction of the underlying axis bandwidth to use

Eg: Assuming x == .5, bandwidth == .8 and the actual underlying axis bandwidth is 40px the width at the given y would be (40 * .8) * .5 or 16px wide

Architecture Notes

Flextool

This is a debug tool in the examples repository which visualizes the bounding boxes for a new or existing template.  The container width and height are a function of the viewport and as such will update the bounding boxes as the geometry of the viewport changes.

Modularization

Core code makes use of import() to dynamically import only those core components and plugins which are needed.  This makes top level invocations of the visualizer asynchronous.  To modify the graph after it has been rendered, the onStateChange callback should be used.

Note: External dependencies are not included by import().  An example of this would be the Tippy(popup) library which would still be loaded manually.

Publish Subscriber Model

Events make use of a publish/subscriber model via PubSubJS.  All UI events will send a tab-wide message to the group specified in the event config.  Every group a visualization would publish it also subscribes to.

Popups

The tippy library has been used to manage all popups in a non-renderer specific way. Specifically, all popups will attach to an absolutely positioned div within the relatively positioned container.  When a pubsub event message (Eg: mouseenter) is received, the eventing element’s position is determined by the renderer, the div is positioned, and the popup shown.  The exception to this is for those plugins that also generate zero radius circles using the same element id+”-alt”.  For these plugins the ‘-alt’ element is used for div positioning instead.  This is relevant for plugins where a centroid derived position (as determined by a bounding box) is non-optimal. Eg: Arc, Rectangle

Caveats

Three Renderer:

Any fonts referenced in the config must exist as .ttf files in ./fonts.

The SVG loader disregards all alpha components of colors.  Specify alpha separately.

The event used for click is mousedown.

Popup points in objects are disregarded.  Mouse location is used instead.

Locally implemented drag and zoom are orthogonal to the same functionality in orbitControls.

Three SVG export does not support textures.  Gradients will be affected.

Currently implemented, lines using dasharray are orthogonal to using textures.

All Renderers:

Not all objects can be scaled correctly.  Some objects not created at the origin might also be translated when scaled.   Circles and points will scale appropriately.

Font attrs must be explicitly assigned to element.

Fonts must be added to constructor call.
